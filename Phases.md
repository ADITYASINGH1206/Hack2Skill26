Phase 1: Data Ingestion & The "Trap" Filter (Offline)Your first goal is to clean the dataset and drop candidates who will get you disqualified or penalized.Stream the Data: Build a Python generator to read candidates.jsonl.gz line-by-line so you don't blow up your RAM.Honeypot Purge: Implement strict logic to drop candidates with impossible profiles (e.g., overlapping timelines or 0 months of duration on "expert" skills). Remember: If your top 100 has >10% honeypots, you are disqualified.  Role Mismatch Filter: Drop "keyword stuffers" (e.g., someone titled "Marketing Manager" who lists "RAG" as a skill).  Logistical Filter: Drop candidates who are pure researchers, pure IT consultants without product experience, or have notice periods well over 30–60 days if they aren't stellar.  Text Extraction: For surviving candidates, concatenate the headline, summary, and the description of their recent roles. Save this to a local clean_pool.jsonl file.  



Phase 2: Embedding & Indexing (Offline)This phase has no time limit. You will generate the artifacts that your fast inference script will load later.Generate Embeddings: Run your clean_pool.jsonl text through a lightweight, open-source embedding model (like all-MiniLM-L6-v2 or bge-small-en-v1.5).Build the Vector Index: Create a dense vector index using FAISS or numpy matrices.  Build the Sparse Index: Create a BM25 index to capture exact keyword matches.  Save Artifacts: Save your dense index, sparse index, and clean metadata to disk. Ensure these artifacts stay under the 5 GB disk limit.  


Phase 3: The Scoring & Reasoning Engine (Runtime Prep)Here, you design the mathematical logic that ranks the candidates.Semantic Score: Calculate a hybrid score comparing the candidate's vector and BM25 score against the Job Description.  Behavioral Multiplier: Parse the redrob_signals. Penalize candidates with a low recruiter_response_rate, no recent last_active_date, or low interview_completion_rate. Multiply your Semantic Score by this Behavioral factor.  Rule-Based Reasoning Generator: Write a fast Python function that dynamically generates a 1-2 sentence string explaining the rank (e.g., "Ranked highly for 6 years of product ML experience and a 90% recruiter response rate"). You cannot use an LLM API for this during runtime, and you must not hallucinate skills.  

Phase 4: The 5-Minute Inference Script (The Core Deliverable)This is the rank.py script the judges will actually execute. It must run completely offline on a 16GB RAM CPU in under 5 minutes.  Load Artifacts: Load your FAISS index and metadata into memory.Execute Scoring: Run the Phase 3 scoring engine on the hidden test pool.Sort and Slice: Sort the candidates by descending score and take exactly the top 100. If scores tie, break them by sorting candidate_id in ascending order.  Format CSV: Output the participant_id.csv with exactly 101 rows (1 header + 100 data rows) and the columns: candidate_id,rank,score,reasoning.

Build the Sandbox: Deploy a small, working version of your ranker (handling ~50-100 candidates) to HuggingFace Spaces, Streamlit Cloud, or a public Replit.
